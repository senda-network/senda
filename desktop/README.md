# ClosedMesh desktop shell

Cross-platform native shell for the ClosedMesh chat UI. One Rust codebase,
three platform bundles:

| Platform | Output                              | Size     | Backing webview |
| -------- | ----------------------------------- | -------- | --------------- |
| macOS    | `ClosedMesh.app` + `.dmg`           | ~5 MB    | WKWebView       |
| Windows  | `ClosedMesh.msi` (+ NSIS `.exe`)    | ~6–8 MB  | WebView2 (Edge) |
| Linux    | `ClosedMesh.AppImage` + `.deb`      | ~10 MB   | WebKitGTK       |

Built on [Tauri 2](https://tauri.app). The shell does not bundle the
runtime; the `closedmesh` CLI is still installed separately via the curl /
PowerShell one-liners in the root README. The desktop app gives the user:

- A real Dock / Start Menu / Taskbar icon and ⌘Tab / Alt-Tab target.
- A system tray pill with live node count, polled from the local admin
  endpoint at `http://127.0.0.1:3131/api/status`.
- Tray menu actions: **Open Chat**, **Reload**, **Open Admin Console**,
  **Start / Stop ClosedMesh Service** (shells out to `closedmesh service
  start|stop`), **Copy Invite Token** (reads the local node's join token
  from `http://127.0.0.1:3131/api/status` and copies it to the system
  clipboard), **Show Logs in File Manager**.
- A WKWebView / WebView2 / WebKitGTK that loads the bundled local
  controller, falling back to
  `https://closedmesh.com` otherwise.

## Develop

Requirements:

- Rust 1.77+ (`rustup default stable`)
- Node 18+ (only used to invoke the Tauri CLI)
- Platform-specific deps for the webview:
  - macOS: nothing extra (WKWebView ships with the system).
  - Windows: WebView2 Runtime (preinstalled on Windows 11; auto-installs
    on Windows 10).
  - Linux: `libwebkit2gtk-4.1-dev`, `libgtk-3-dev`, plus `libsoup-3.0-dev`
    on most distros (`apt install libwebkit2gtk-4.1-dev libgtk-3-dev libsoup-3.0-dev`).

```bash
cd desktop
npm install            # one-time: installs @tauri-apps/cli + @resvg/resvg-js
npm run icons          # rasterize source.svg -> per-platform icon variants
npm run dev            # hot-reload dev build (cargo + tauri)
```

`npm run dev` will rebuild on Rust file changes; the WebView itself loads
the bundled local controller / `closedmesh.com`, so frontend changes are
picked up by reloading.

## Build distributable bundles

```bash
./scripts/build.sh            # release build for the host OS
./scripts/build.sh --debug    # debug profile, ~3x faster compile
./scripts/build.sh --skip-icons   # don't regenerate icons
```

Output goes to `target/release/bundle/`:

```
target/release/bundle/
  macos/ClosedMesh.app
  dmg/ClosedMesh_<version>_<arch>.dmg
  # Windows builds add: msi/, nsis/
  # Linux builds add: deb/, appimage/
```

The bundles are **unsigned**. On macOS that means a one-time "right-click →
Open → Open" the first time a teammate runs the .app; on Windows, SmartScreen
will warn until we register a Developer ID. For wider distribution we'll
want:

- macOS: $99/yr Apple Developer ID + `codesign --options runtime` + notarization.
- Windows: $200–$500/yr code signing certificate (or use SignTool with a
  self-signed cert for internal-only).
- Linux: detached `.sig` files alongside the `.AppImage` are usually enough.

These are tracked but out of scope for the initial cut.

## Architecture

```
desktop/
  Cargo.toml             Rust deps. Tauri 2 + ureq (status polling) +
                         arboard (clipboard) + dirs (log dir lookup).
  tauri.conf.json        Bundle config (identifiers, bundle targets,
                         platform-specific bits like macOS minimum
                         version, Linux .deb depends).
  build.rs               One line: tauri_build::build().
  src/
    main.rs              App entry, window creation, tray builder,
                         menu event dispatch, status polling loop.
    mesh.rs              Talks to localhost:3131/api/status via ureq;
                         shells out to the closedmesh CLI for service
                         start/stop/invite. Cross-platform binary
                         discovery (~/.local/bin, /opt/homebrew, $PATH,
                         %LOCALAPPDATA%, etc).
  icons/
    source.svg           Design source-of-truth (3-node mesh glyph,
                         matches app/components/Logo.tsx — same orange
                         accent colour, #ff7a45, kept in sync with
                         app/globals.css).
    source.png           Generated by scripts/svg-to-png.mjs (gitignored).
    {32x32,128x128,...}.png, icon.icns, icon.ico
                         Generated by `tauri icon`. All gitignored —
                         regenerated on every build.
  scripts/
    build.sh             SVG -> PNG -> tauri icon -> tauri build.
    svg-to-png.mjs       Pure-Rust SVG rasterizer (@resvg/resvg-js).
                         Same script runs on macOS / Linux / Windows CI.
                         Emits desktop's icons/source.png AND the web
                         favicon set (../app/icon.svg, ../app/apple-icon.png,
                         ../public/icon-{192,512}.png). The wrapping
                         `npm run icons` script also copies the generated
                         icon.ico into ../app/favicon.ico so the website
                         and the desktop app share one branded glyph.
  dist/
    index.html           Splash shown for ~100ms before the WebView
                         navigates to the chat URL. Tauri's bundler
                         requires *some* frontend dist; this is it.
```

## Why this replaced the old `mac-app/` (Swift)

Earlier iterations had a separate Swift / AppKit Mac app (`mac-app/`).
We removed it after one release because:

- Tauri uses the same WKWebView on macOS that AppKit does, so the user
  experience on Mac doesn't degrade — same backing webview, same ~5MB
  bundle, same native menu bar.
- Maintaining one cross-platform GUI codebase is easier than three.
- The Rust toolchain is already required for the runtime
  (`closedmesh-llm`), so Tauri doesn't add a new language to the project.

The Swift implementation is preserved in git history at the commit before
this README was added; check `git log -- mac-app/` to find it.
